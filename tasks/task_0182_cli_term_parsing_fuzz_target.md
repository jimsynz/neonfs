# Task 0182: CLI Term Parsing Fuzz Target

## Status
Blocked — depends on 0179 (fuzz scaffolding); needs CI design

## Phase
Gap Analysis — H-3 (4/5)

## Description
Create a fuzz target for the `neonfs-cli` Rust crate that exercises Erlang
term parsing. The CLI receives responses from the Elixir daemon as encoded
Erlang terms (`eetf` crate) — malformed terms must be handled gracefully.

This is a separate fuzz workspace from the `neonfs_blob` targets since it
lives in a different crate.

## Notes
The CLI's term parsing is a security-sensitive boundary: the daemon sends
Erlang-encoded responses over a TCP connection. If an attacker can inject
malformed terms, the CLI must not crash or exhibit undefined behaviour.

The seed corpus should be generated by encoding terms from Elixir:

```elixir
# Generate seed corpus files
:erlang.term_to_binary({:ok, "result"}) |> File.write!("corpus/ok_result")
:erlang.term_to_binary({:error, :not_found}) |> File.write!("corpus/error_atom")
:erlang.term_to_binary({:error, %{__struct__: NeonFS.Error, message: "test"}})
|> File.write!("corpus/error_struct")
```

The fuzz target structure:

```rust
fuzz_target!(|data: &[u8]| {
    // Try decoding as Erlang term
    if let Ok(term) = eetf::Term::decode(data) {
        // Try extracting as a response
        let _ = extract_response(&term);
        // Try extracting as an error
        let _ = extract_error(&term);
    }
});
```
