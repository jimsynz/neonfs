# Task 0176: HTTP Metrics Endpoint (Bandit + Plug)

## Status
Complete

## Phase
Gap Analysis — H-2 (4/6)

## Description
Add an HTTP server (Bandit + Plug) to `neonfs_core` that serves two
endpoints:
- `GET /metrics` — Prometheus text exposition format
- `GET /health` — JSON health status from `HealthCheck`

The server listens on a configurable port (default 9568) and is intended
for scraping by Prometheus and health checking by orchestrators.

## Notes
Bandit is preferred over Cowboy for its lightweight nature and pure-Elixir
implementation. The metrics server is minimal — just two routes.

The Plug implementation should be simple:

```elixir
defmodule NeonFS.Core.MetricsPlug do
  use Plug.Router
  plug :match
  plug :dispatch

  get "/metrics" do
    # ...
  end

  get "/health" do
    # ...
  end

  match _ do
    send_resp(conn, 404, "Not Found")
  end
end
```

Port 9568 is a common convention for Prometheus exporters (not in the
registered port range, avoids conflicts with common services).

The health endpoint returning 200 for degraded (not 503) is intentional —
degraded means the node is functional but may have reduced capacity. Load
balancers should only remove truly unhealthy nodes.
